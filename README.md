# -motivation-streaming-app

## サービス概要
配信活動に悩む配信者向けに、分析シートで悩みを構造化し、AIが配信者特化の分析で最適な目標と行動プランを提案するWebアプリです。
ChatGPTとは異なり「何を相談すればいいかわからない」を解消し、視聴者数・収益化・モチベーション維持など配信者特有の課題に対応します。
過去の記録から成長を実感でき、目標が変化しても継続的にサポートし、必要なリソースや仲間との出会いも創出します。

## このサービスへの思い・作りたい理由
5年間の配信活動の中で、私は多くの配信者が半年〜1年で辞めていく姿を見てきました。
私自身も葛藤を抱えながらの活動でしたが、現状を分析し適切な目標を設定することで実績を伸ばすことができました。「続けたいのに辞めてしまう」配信者たちに足りないのは、課題の整理と目標設定のサポートだと気づきました。
配信活動を通じて、自分が培ってきた分析力と課題解決力が配信者の悩みに応用できると確信しました。この強みを活かし、同じ悩みを持つ配信者の継続をサポートしたい。その思いがこのサービスを作る原動力となっています。

## ユーザー層について
**ターゲット層:**
続けたいけど、挫折しそうなモチベーション維持に悩む配信者

**理由:**
配信活動は始めやすい反面、継続が非常に難しいという特徴があります。5年間の活動を通じて、多くの配信者が半年〜1年でモチベーション維持に悩み、やめていく姿を見てきました。

私自身も同じ悩みを抱えながら活動を続けてきたため、この層が抱える課題を最も深く理解しています。「続けたい気持ちはあるのに、何をすればいいか分からない」「目標が見えず、モチベーションが保てない」といった悩みは、まさに自分が経験してきたものです。

実体験に基づいた実践的なサポートを提供できることが、このターゲット層を選んだ最大の理由です。

## サービスの利用イメージ
配信活動で「何をすればいいか分からない」「モチベーションが続かない」と悩んだとき、スマホやPCから気軽にアクセスし、自分の配信の現状と心境を整理できます。

このサービスでは、配信者が抱える課題を分析し、具体的で実現可能な目標設定をサポートします。さらに、目標達成に必要な情報やツール、サービスを紹介することで、一人で悩まず、前に進める環境を提供します。

## サービスの差別化ポイント・推しポイント

### 1. 類似サービスとの違い

#### 習慣化サービス(マイルーティン、継続する技術など)との違い

**類似サービスの特徴:**
- 習慣化全般をサポート
- 毎日の記録が前提
- 記録・可視化が中心

**当サービスの差別化ポイント:**

1. **配信者の活動継続に特化**
   - 習慣化全般ではなく、配信者特有の悩み(視聴者数、収益化、モチベーション維持など)に焦点を当てた分析基準を持つ

2. **月1回の記録で十分な気軽さ**
   - 毎日の記録は不要。困ったときに気軽に利用できる
   - 配信活動に集中しながら、必要なときにサポートを受けられる

3. **AIによる個別アドバイスと具体的なアクションプラン**
   - 単なる記録・可視化にとどまらず、AIが配信者の状況を分析し、適切な目標設定と具体的なアクションプランを提案
   - 「なぜモチベーションが低下したのか」を分析し、同じ失敗を繰り返さないためのヒントを提供

---

### 2. ChatGPTとの違い

#### 「何を相談すればいいかわからない」層への対応

**ChatGPTの課題:**
- 自由度が高い反面、「どう質問すればいいかわからない」という壁があります。
- 汎用的なアドバイスになりがちです。

**当サービスの強み:**

1. **分析シートで質問を構造化**
   - 悩みを言語化しやすくし、相談までのハードルを大幅に下げる
   - ChatGPTでは届かない「何を相談すればいいかわからない」層にリーチできる

2. **配信者特化の分析基準**
   - 配信者特有の悩み(視聴者数、収益化、モチベーション維持など)に特化した分析基準を持つ
   - 入力された分析シートをもとに、各配信者が目指す配信の形に対して、現状の改善ポイントを具体的に提案

3. **過去の記録を活用した振り返り機能**
   - ChatGPTは会話履歴が残るが、振り返りには向いていない
   - 本サービスでは過去の分析シートを保存し、モチベーションが下がったときに見返せる
   - 自分の成長を可視化し、同じ悩みを繰り返さないようサポート

---ChatGPTは「その場限りの相談相手」で当サービスは「継続的な成長パートナー」になります。

### 3. 継続利用の価値

#### 配信者の目指す配信の形は変化し続ける

配信を続ける中で、目指す配信スタイルや目標は常に変化します。

**目標変化の例:**
- 「雑談配信で楽しみたい」→「収益化を目指したい」→「特定のゲームで有名になりたい」

**本サービスが提供する継続的な価値:**

1. **悩みの可視化と解決手順の提示**
   - 漠然とした悩みを具体的な課題として認識できる
   - その課題を解決するための手順・ツール・情報を自分で調べなくても得られる
   - 調べるのが苦手な人でも、適切な情報に迷わずたどり着ける

2. **過去との比較による成長の実感**
   - 過去の分析シートを見返すことで、自分がどれだけ成長したかを実感できる
   - モチベーションが下がったときに、過去にどんな壁にぶつかり、どう乗り越えたのかを思い出せる(同じ課題に直面しても、過去の解決策を参考にできる)
   - 目標が変化した際に、過去の自分と今の自分を比較し、次のステップを明確にできる

3. **変化する目標に合わせた柔軟なサポート**
   - 配信者によって目的や「楽しい」「悲しい」と思うポイントは異なる
   - さらに、同じ配信者でも目標は変化し続ける
   - 本サービスでは、その時々の目標に合わせたアドバイスを提供し、画一的な解決策ではなく、今のあなたに最適な方法を提案

---

### 4. AIとメンター・コミュニティの役割分担(将来構想)

#### なぜマッチング機能が必要か

配信者の情報収集スタイルは2つに分かれます。

**1. 自分で調べられる層**
- AIが提供した目標と課題分析結果をもとに、自分で適切なサービスやツールを探しに行く
- 本サービスでは、探すための道筋を示すことで、効率的な情報収集をサポート

**2. 調べるのが苦手・不得意な層**
- どう検索すればいいかわからない、忙しくて調べる時間がない、という人が一定層存在
- 実際、SNSでハッシュタグをつけて絵師やミックス師を募集している傾向がある
- 本サービスでは、マッチング機能を提供することで、調べる手間を省き、適切なサポーターと出会える場を作る

#### マッチング機能の流れ

1. 配信者が匿名アカウントで分析シートのお悩みを投稿
2. お悩みを解決できると思った企業やアドバイザーが「いいね!」を押す
3. 配信者側も「この人に助言をもらいたい」と思った相手に「いいね!」を返す
4. マッチング成立でDMが解放され、具体的な相談が可能


#### AIと人の役割分担

**AIの役割:**
- **論理的な分析**: 入力された分析シートをもとに、客観的に課題を特定
- **目標設定**: 配信者の現状と目指す方向性に合わせた、具体的な目標を提案
- **情報提供**: 目標達成に必要なツール・サービス・ノウハウを提示

**人(メンター・コミュニティ)の役割:**
- **実体験に基づくアドバイス**: AIにはできない、「実際にやってみてどうだったか」という先輩配信者の知見を提供
- **人脈形成**: 配信者同士、または配信者とサポーター(絵師、ミックス師、企業など)を繋ぐ
- **必要なサービスとの出会い**: 調べるのが苦手な層に対して、マッチング機能で適切な人・サービスを届ける

**両者を組み合わせることで実現する価値:**
- **調べられる人向け**: AIの分析をもとに、自分で最適なサービスやツールを探せる
- **調べるのが苦手な人向け**: マッチング機能で、自分に合ったサポーターやサービスと出会える
- **すべての配信者向け**: 論理的な分析(AI)と人の温かみ(コミュニティ)の両方を得られ、孤独にならず継続的に成長できる環境が整う

### 推しポイント

**「配信者の"続けたい"を、AIと記録と人で支えるサービス」**

#### 1. 悩みを言語化しやすい「分析シート」
- 「何を相談すればいいかわからない」という悩みを解消
- 月1回の記録で、自分の状況を整理し、漠然とした悩みを具体的な課題に変換

#### 2. 配信者特化のAI分析
- AIが現状に合った**目標とアクションプラン**を提案
- 配信者特有の悩み(視聴者数、収益化、モチベーション維持など)に対応
- モチベーション低下の原因を可視化し、改善をサポート

#### 3. 変化する目標に対応した継続サポート
- 配信者の目指す配信の形は変化し続ける
- 「雑談配信で楽しみたい」→「収益化を目指したい」→「特定のゲームで有名になりたい」など、目標が変わっても対応
- **その時々の目標に合わせた情報・ツール・人脈を提供**し続ける

#### 4. 成長を実感できる記録機能
- 過去の分析シートを見返すことで、**自分がどれだけ成長したか**を実感
- モチベーションが下がったときに、**過去の自分がどう壁を乗り越えたかを思い出せる**

#### 5. 調べるのが苦手な人にも優しいマッチング機能(将来構想)
- 調べられる人には「道筋」を、調べるのが苦手な人には「出会いの場」を提供
- 配信者とサポーター(絵師、ミックス師、企業など)を繋ぐマッチング機能で、適切なリソースと出会える

---

**配信者が成長を感じながら、楽しく配信を継続できるサービスです。**

## 一般的なCRUD以外の実装予定の機能

### 1. AI機能

**使用技術**: OpenAI API (ChatGPT API) + ruby-openai gem

**実装の流れ:**
1. ユーザーが分析シートを入力・保存
2. 「アドバイスを見る」ボタンをクリック
3. サーバー側で入力内容をもとにプロンプトを作成
4. OpenAI APIにリクエストを送信
5. AIが生成したアドバイスをデータベースに保存
6. 画面に表示

**参考記事:**
- [ruby-openai gemの使い方](https://github.com/alexrudall/ruby-openai)
- [OpenAI API公式ドキュメント](https://platform.openai.com/docs/api-reference)

### 2. グラフ・可視化機能

**使用技術**: Chart.js (JavaScript)

**実装の流れ:**
1. 過去の記録をデータベースから取得
2. JSON形式でフロントエンドに渡す
3. Chart.jsを使ってグラフを描画

**表示予定のグラフ:**
- 配信頻度の推移(折れ線グラフ)
- モチベーションの変化(折れ線グラフ)
- 視聴者数の推移(棒グラフ)

**参考記事:**
- [Chart.js公式ドキュメント](https://www.chartjs.org/docs/latest/)
- [RailsでChart.jsを使う方法](https://qiita.com/search?q=Rails+Chart.js)


## 今後の展望

### フェーズ1(MVP): 配信活動の記録・振り返り基盤
まずは、配信者が自身の活動状況やモチベーションを記録し、過去の状態を振り返ることができる基盤を提供します。分析シートに答えるだけで、配信活動の現状を構造化して記録できる仕組みを構築します。

### フェーズ2: AI分析・アドバイス機能の追加
記録した情報をもとに、ChatGPT APIを活用した個別分析・アドバイス機能を実装します。配信者特有の悩みに対応した目標設定の提案や、具体的なアクションプランの提示により、モチベーション維持をより強力にサポートします。

### フェーズ3: コミュニティ・メンター機能
配信活動を続けている先輩配信者が、後輩配信者をサポートするためのメンター機能を追加します。経験者の知見を共有し、コミュニティとしての価値を高めます。

### フェーズ4: 企業・営業職への展開
配信者やクリエイターの情報を蓄積し、企業の営業担当者が適切なタイミングで適切な相手に営業DMを送れるマッチングプラットフォームへと発展させます。

**背景:**
現在の営業活動では、配信者やクリエイターへの営業DMは「迷惑」と捉えられがちです。しかし、配信者が「まさにこんなサービスを探していた!」と感じるタイミングで適切な提案ができれば、営業活動は「価値提供」に変わります。

**具体的な仕組み:**

1. **匿名性の確保**
   - 配信者は匿名アカウントでサービスを利用
   - チャンネル名や実名などの個人情報は非公開
   - ライバルに情報が漏れるリスクを最小化

2. **企業からのアプローチ**
   - 企業は配信者の活動ジャンルや課題感に基づいてDMを送信
   - 配信者の詳細情報は見えない状態でアプローチ

3. **配信者による任意の情報開示**
   - 配信者が興味を持った提案にのみ返信
   - DM内で必要な情報を任意で開示
   - 配信者が主導権を持つ形でのやり取り

**目指す姿:**
営業活動が「押し売り」ではなく「必要な情報を必要な人に届ける」価値提供の場となり、配信者・企業双方にとってWin-Winの関係を構築します。配信者はプライバシーを守りながら有益な情報を得られ、企業は本当に興味を持つ相手とだけ商談を進められます。

**注意:**
フェーズ4は長期的な展望であり、まずはフェーズ2・3で配信者コミュニティを確立し、十分なデータと信頼を蓄積することが前提となります。

**プライバシーへの配慮:**
- 配信者が明示的に「営業を受けたい」と設定した場合のみ情報を共有
- 配信者側で受け取りたい営業のジャンルを選択可能
- いつでもオプトアウト可能
